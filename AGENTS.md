# CartWise 开发规则

CartWise 是一个基于 RAG 的可解释个性化电商导购系统。

- 严格按照 `docs/DEVELOPMENT_STEPS.md` 逐阶段开发。
- 每次只完成当前阶段，不要提前扩展后续功能。
- 从仓库根目录运行命令，使用 `.\.venv\Scripts\python.exe`。
- 下载依赖、模型和数据时使用代理 `http://127.0.0.1:9508`。
- 访问 `127.0.0.1` 和 `localhost` 时绕过代理。
- 禁止批量删除文件或目录；如需大量清理，让用户手动执行。
- 不要覆盖或撤销无关改动。
- 不要提交 `.env`、数据、索引、模型权重或真实 API Key。
- 修改文件或提交代码前检查 `git status`。
- 每次开发前先结合已有模块代码设计实现；优先复用现有接口、工具函数和数据口径，避免重复实现，保持模块职责明确、依赖关系清晰和整体结构易于维护。
- 新增召回通道时必须接入统一召回审核工具，并支持生成供人工审核的 HTML 和机器可读的 JSON 报告。
- 每次对话中发现适合长期复用的类似规则时，先询问用户是否写入 `AGENTS.md`。
- Dense 召回默认使用 E5。
